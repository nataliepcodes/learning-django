To deactivate: $ deactivate

# Install Django
Documentation: https://docs.djangoproject.com/en/6.0/topics/install/#installing-official-release
$ python3 -m pip install Django

# Check that installation has been successful
$ python
>>> import django
>>> django.get_version()

NOTE: if the output of the django version is not the latest official version:
-- check the python version: $ python3 and $ python
-- reinstall everything again, if necessary
-- to delete a directory with all files: $ rm -r directory_name

# Project vs application
In Django a website would be called a project. 
Applications are some parts of the website - e.g. blog, wiki, or forum.
The applications can be used by other projects
To create a Blog application: python manage.py startapp blog

# Data Models
Model is a Python class that subclasses django.db.models.Model where each attribute is a database field
A table is created for each model in models.py file
Djago provides various fields for data models: https://docs.djangoproject.com/en/6.0/ref/models/fields/
Custome model fields can also be created: https://docs.djangoproject.com/en/6.0/howto/custom-model-fields/

# Activate the application
Django needs to create the database tables for models and to track the application. 
For that - the application needs to be activated
To activate:
-- edit settings.py file by adding blog.apps.BlogConfig to the INSTALLED_APPS

# Create a database table for the Model
!!Reminder!!: In Django a Website is a Project, a Blog is an App, a Blog Post is a data Model
1) Create initial migration for the Post data model, in the root dir: $ python manage.py makemigrations blog
^^ for migration to work, ensure that the app is activated, otherwise I will get an error: No installed app with label 'blog'.
^^ when the initial migration is done, a file is created in the blog directory, the file is called 0001_initial.py
^^ 0001_initial.py specifies dependencies and operations within the Migration class
2) Run this to inspect the output: $ python manage.py sqlmigrate blog 0001
3) Sync the db with the new data model: $ python manage.py migrate

# Creating administration site to manage blog posts
Django has a build-in admin interface, use out of box or can be configured to suit
INSTALLED_APPS already includes django.contrib.admin
1) Creating a superuser who will manage the admin site: 
1-1) $ python manage.py createsuperuser
1-2) Start development server: $ python manage.py runserver
1-3) Open: http://127.0.0.1:8000/admin
1-4) Log in and Be super happy that it all worked well :)
