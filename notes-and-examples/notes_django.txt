1) Creating a superuser who will manage the admin site: 
1-1) $ python manage.py createsuperuser
1-2) Start development server: $ python manage.py runserver
1-3) Open: http://127.0.0.1:8000/admin
1-4) Log in and Be super happy that it all worked well :)

# Database management
Django comes with db abstraction API which helps to create, retrieve, delete and update objects
ORM/Object-Relational-Mapper is compatible with MySQL, PostgreSQL, SQLite, Oracle and MariaDB
Django ORM is based on QuerySets that are requeries to get objects from the database
Database for the project can be identified in settings.py under DATABASES
DOCS:
https://docs.djangoproject.com/en/6.0/ref/models/
https://docs.djangoproject.com/en/6.0/topics/db/models/
"objects" is a default manager of each model. "object" gets all objects in the database. it can be customised in models.py
To retrieve objects: Post.objects.all()
To filter: Post.objects.filter(publish_year=2026, author='admin')
To exlude: Post.objects.filter(publish_year=2026).exclude(title_startswith='Who')
To delete: 
>>> post = Post.objects.get(id=8)
>>> post.delete()


# Building views of blog application
In Django a view is a Python function. It receives a web request and returns a web response.
'View' contains the logic of the response. It renders a template with variables passed to it. 
The view returns HTTP response together with the rendered output. The views are created in views.py inside blog post application
Steps:
-- Create application views
-- Define URL pattern for each view
    * URL patterns is made of a string patter, a view and as an option a name that can be used across the whole project
    * Django runs through each URL pattern, if requested URL matches then it stops at that URL
    * After that, "Django imports the view of the matching URL pattern, executes it, passes an instance of HttpRequest class and the keyword or positional argument" (p.28)
    * Canonical URLs are used as the main URL for the model (e.g. Blog post), get_absolute_url() are added to the model
-- Create HTML templates to render data that views generate
    * templates structure:
    templates/
        blog/
        base.html
            post/
                list.html
                detail.html
    * templates are written in HTML and Django templates language
    https://docs.djangoproject.com/en/6.0/ref/templates/language/

>> How to send emails with Django?
Option 1: Have a local SMTP server, or 

Option 2: Add the below external SMTP server configuration into the settings.py
EMAIL_HOST:localhost
EMAIL_PORT: 587
EMAIL_HOST_USER: 'SMTP server username'
EMAIL_HOST_PASSWORD: 'SMTP server password'
EMAIL_USE_TLS: True or False
or
EMAIL_USE_SSL: True or False

Option 3: If SMTP server can't be used, emails can be written to console, add below to settings.py
# Write emails to the console (without SMTP) for TESTING
EMAIL_BACKEND = "django.core.email.backends.console.EmailBackend"

Option 4: Use SMTP server of the email sevice provider, this can be configured, for example, via a Google account

# BLOG FEATURE: Full-text search with PostgreSQL
- Simple search can be performed with Django ORM 'contains' filter
- Full-text search requires PostgreSQL
- install PostgreSQL >> https://www.enterprisedb.com/downloads/postgres-postgresql-downloads
- install psycopg2 PostgreSQL adaptor for Python >> pip install psycopg2-binary
- create user (on MacOS):
    >> psql -U postgres
    >> Password for user ***
    !! 'postgres=#' means Successful connection to PostgreSQL
    >> postgres=# CREATE USER blog WITH PASSWORD 'strongpassword' CREATEDB;
    >> postgres-# CREATE DATABASE blog OWNER blog;
    !! check status with \du or \l
    !! CTRL + D to exit. Don't quit - exit and continue :)

- edit settings.py DATABASES 
    !! When changing DB 'ENGINE' from sqlite to PostgreSQL, NOTE that all sqlite db data will disappear from admin view - migrate/backup what's necessary
- run $ python manage.py migrate
- python manage.py createsuperuser
- Building a search view:
    -- Create a search form in forms.py
    -- Edit views.py by adding post search function
    -- Create a template to display the form and the results
    -- Edit urls.py by adding the search URL pattern
- Processing results    
    -- Stemming is a process to reduce words to their word "stem, base or root phase" - page 88
    -- PostgreSQL has a ranking function based on frequency of the query term and how close/similar these are
    -- Edit views.py to SearchQuery and SearchRank
- What's a Vector?:
    -- vector has different meanings in programming: dynamically sized array, data structure etc
    -- https://www.paulox.net/2017/12/22/full-text-search-in-django-with-postgresql/#searchvector
    -- In simple terms, the SearchVector (in the context of mysite blog project) is where the data is SEARCHED FROM - for example, FROM the body of the text, or FROM the title
        - search_vector = SearchVector('title', 'body')
        - search_query = SearchQuery(query)
        - a query is WHAT we search for, and a vector is WHERE
- Trigram similarity search
    -- trigram: grouping of 3 sequential elements
        > in Linguistics: A sequence of three adjacent letters, symbols, or words in a text (also known as a trigraph or 3-gram)
        > in Computer Science: In Natural Language Processing (NLP), a 3-letter or 3-word sequence used for text analysis, speech recognition, and spelling correction
    -- Common Uses:
        > Search Optimization: The PostgreSQL pg_trgm  databases quickly find misspelled search queries
        https://www.postgresql.org/docs/current/pgtrgm.html
        > Predictive Text: A trigram model predicts the probability of the next item based entirely on the two items that came before it
        https://en.wikipedia.org/wiki/Trigram_search
    -- install PostgreSQL pg_trgm extention
        Check if 'pg_trgm' exists:
        1.  BASH: $ python manage.py dbshell
        2.  SQL: SELECT current_database();
        3.  SQL: SELECT extname FROM pg_extension WHERE extname = 'pg_trgm';
        If zero rows, then:
        1.  BASH: $ python manage.py dbshell
        2. SQL: CREATE EXTENSION IF NOT EXISTS pg_trgm;
        3. SQL to verify: SELECT extname FROM pg_extension WHERE extname = 'pg_trgm';
    -- edit views.py to include TrigramSimilarity and updated Post search query
